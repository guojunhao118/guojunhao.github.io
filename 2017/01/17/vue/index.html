简介

vue的优势是什么?(vue可替换成angular,react)
代码质量高 开发速度快 库多 库质量高 效率高 简单易用
//组件化开发
jquery 这种处理方式在业务复杂的情况下，复杂程度急剧增加
而 vue 采用了 mvvm 的方式，双向绑定，可以是我们专注业务逻辑
所以开发更快
前端的逻辑不同后端，组件化开发是前端的未来，而 vue 学习曲线平缓，收益可观同时 jquery 是命令式编程，而 vue 采用的是声明式，前端的 HTML CSS 都是声明式的，JS 为啥不能采用的。这样我们开发会更快，debug 的时候也会更加方便

vuex

视频 26 分开始是 vuex

  • store

    从 store 实例中读取状态最简单的方法就是在计算属性中返回某个状态
    ·使用方法 => store.xxx

  • getter

    就像计算属性一样，getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算
    ·使用方法 => store.geeters.xxx
    Getter 也可以接受其他 getter 作为第二个参数：

  • mutation

    mutation 中定义方法对 state 数据进行操作

    ·使用方法 => store.commit(‘name’, payload)

    大多数情况下，载荷（payload）应该是一个对象，这样可以包含多个字段并且记录的 mutation 会更加易读

    mutation 必须是同步函数

  • action

    Action 提交的是 mutation，而不是直接变更状态。

    Action 可以包含任意异步操作

    Action 函数接受一个与 store 实例具有相同方法和属性的 context 对象，因此你可以调用 context.commit 提交一个 mutation，或者通过 context.state 和 context.getters 来获取 state 和 getters

    使用方法 => store.dispatch

  • module

vuex的几种状态
|函数|用法|说明|
|:–:|:–:|:–:|
|mutations|this.$store.commit()|这是操作状态的手段|
|actions|this.$store.dispatch()|其中可以进行异步操作|
