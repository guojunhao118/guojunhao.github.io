选择器&权重

举例：<h1 id=”id-h1“ class=”class-h1“> 优先级 < /h1>
权重记忆口诀：从0开始，一个行内样式+1000，一个id+100，一个属性选择器/class或者伪类+10，一个元素名，或者伪元素+1

元素选择器 符号 [优先级从高到低][4]

class选择器 . .class-h1{color:red;}
标签选择器 直接选择 h1{color:red;}

关于position定位

值 描述
relative 相对定位，相对于它的默认位置移动，不会影响页面布局(还会占有原来的空间)
absolute 绝对定位，使用绝对定位，必须先在父元素（不能是static）设置相对定位，脱离了文档流
fixed 相对于窗口来定位,所以它永远不会移动，一般设置top,left属性
inherit 规定应该从父元素继承 position 属性的值
static 位于正常的文档流中，由浏览器控制位置，无法设置top等属性

非 static 元素可以用 top left bottom right 属性来设置坐标
非 static 元素可以用 z-index 属性来设置显示层次
absolute 完全绝对定位, 忽略其他所有东西往上浮动到 非 static 的元素

    .css {

    -webkit-transform-style: preserve-3d;
-webkit-backface-visibility: hidden;
-webkit-perspective: 1000;

    }
    过渡动画（在没有启动硬件加速的情况下）会出现抖动的现象， 以上的 解决方案只是改变 视角 来启动硬件加速的一种方式；
    启动硬件加速的 另外一种方式： 

    .css {
    -webkit-transform: translate3d(0,0,0);
    -moz-transform: translate3d(0,0,0);
    -ms-transform: translate3d(0,0,0);
    transform: translate3d(0,0,0);
}

    启动硬件加速
    最常用的方式：translate3d、translateZ、transform

    opacity属性/过渡动画（需要动画执行的过程中才会创建合成层，动画没有开始或结束后元素还会回到之前的状态）

    will-chang属性（这个比较偏僻），一般配合opacity与translate使用（而且经测试，除了上述可以引发硬件加速的属性外，
    其它属性并不会变成复合层），

    弊端： 硬件加速会导致 CPU性能占用量过大，电池电量消耗加大 ；因此 尽量避免泛滥使用硬件加速。

代码原理

这是阿里团队的高清方案布局代码，所谓高清方案就是根据设备屏幕的DPR（设备像素比，又称DPPX，比如dpr=2时，表示1个CSS像素由4个物理像素点组成） 动态设置 html 的font-size, 同时根据设备DPR调整页面的缩放值，进而达到高清效果。
