JavaScript

入门: Head First HTML5 Programming（中文版）

第 1 到 4 章是 JS 的基础知识，多花些时间看看。后面章节只要是 HTML5 新增的 API，可以快速过一遍，感兴趣多翻翻也好，期间多参考配套代码 bethrobson/Head-First-HTML5

JavaScript高级程序设计（第3版）

红宝书阅读指南
● 第一章随便看看，了解历史
● 第二章简单看看，script 里的 defer 和 async 属性了解下就好了，noscript 不用看
● 第三章位运算、label、with 不用看
● 第四章概念性质的了解就可以，基本类型与引用类型需要掌握
● 第五章正则不用看/page116(134)
● 第六章都看看，重点内容
● 第七章也是重点内容，都需要看
● 第八章了解就可以了，知道 bom是什么，用的时候能查到
● 第九章随便看看，有个印象就行 (用户代理检测一节没有细看)
● 10，11，12 了解 DOM 的类型，掌握操作 DOM，包括选择、style、文本等
● 13 事件中和 IE 有关的部分不用看
● 14 表单内容需要掌握，可以和 jquery 的 api 结合看，简化理解
● ==15 可以跳过，有兴趣的自己看看就好了==
● 16 了解就行了，postMessage 可能会出现在跨域问题中
● ==17 18 19 跳过就可以了==
● 20 21 是 JSON 和 AJAX，不过大部分应该掌握了，新加入的东西看看好了，用的时候
● 比如 formData 在上传文件的时候需要用上，progress 用的比较少。
● 跨域知识比较旧，但是面试的套路就是要掌握这些，所以还是需要看。
● ==22 高级技巧感兴趣就看看，因为要求相对较高，不愿意看也没有关系==
● ==23 看看 localStorage 和 cookie 就可以，其他的实际用得不多==
● ==24 看看还是有必要的，算是合格的编程风格==
● ==25 使用比较多的是 requestAnimationFrame（主要是改良动画） 和 File API，其他的使用较少，用的时候再查就可以了==
● 附录部分里的严格模式不用看，其他了解下就可以

###《JavaScript 面向对象精要》

还是 Zakas 的书，很薄的一本小册子，讲述了 JavaScript 面向对象的知识，算是 JavaScript 高级程序设计 5 6 7 三章的补充。

《JavaScript 语言精粹》

老道的书，算是他个人理解的 JavaScript 最佳实践，书里面最后的 JSON Parser 算是彩蛋。我觉得这本应该和《Effective JavaScript》一起看的，当然后者的内容相对较新。

《JavaScript 忍者秘籍》

jQuery 作者的书，以前相对讲 JavaScript 比较高级的书。可惜出得比较晚，有些内容现在看来比较过时，有点土法炼钢的味道。
